# Schema Versioning

> Language: English | [中文简体](./zh-CN/03-reference/schema-versioning.md)

AgentScope trace JSON now uses a stable `traceVersion` field.

## Current Version

The current stable trace schema version is:

```text
1.0
```

New AgentScope traces generated by the browser importers, live-capture CLI, sanitized exports, and static reports should include:

```json
{
  "traceVersion": "1.0"
}
```

## Compatibility

Importers normalize older unversioned traces to `traceVersion: "1.0"` when loading them. This keeps existing sample files and older local captures viewable while giving new tools a stable version marker.

## Change Policy

Patch-level importer improvements should keep `traceVersion` unchanged when the JSON shape remains compatible.

Use a new schema version only when a trace producer or consumer must change behavior because fields are removed, renamed, or materially reinterpreted.

## Validation

Run the importer fixture test suite before changing schema behavior:

```bash
npm test
```

The test suite verifies supported fixtures, required step fields, importer-specific metadata, tool-result pairing, token extraction, and `traceVersion`.
